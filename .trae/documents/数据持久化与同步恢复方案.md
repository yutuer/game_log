# 数据持久化与同步恢复方案（基于 Log4j2）

## Summary
使用 Log4j2 框架实现日志数据双重保障：1) 请求入口通过专用 Logger 写入滚动文件；2) 启动时检测并同步未入库的数据。

## Current State Analysis

### 现有架构
- **日志框架**: Spring Boot 默认 Logback（application.yml 配置）
- **异步写入**: `GameLogAsyncWriter` 使用内存队列 `LinkedBlockingQueue`
- **问题**: 服务崩溃时队列数据丢失

---

## Proposed Changes

### Step 1: 添加 Log4j2 依赖

**文件**: `pom.xml`

- 排除 spring-boot-starter-logging
- 添加 log4j2-core, log4j2-spring-boot-starter
- 版本统一管理

### Step 2: 创建 Log4j2 配置文件

**文件**: `src/main/resources/log4j2-spring.xml`

- 配置 RollingFile Appender（按天滚动）
- 使用 JSON 格式写入数据
- 文件路径: `logs/data/game-log-{date}.json`
- 保留最近 7 天

### Step 3: 创建专用 Logger 组件

**文件**: `src/main/java/com/gamelog/async/DataLogWriter.java`

- 使用 Log4j2 的 Logger（专用 logger name）
- 提供 `logData(GameLog)` 方法异步写入
- 不阻塞主业务流程

### Step 4: 集成到异步写入流程

**修改**: `GameLogAsyncWriter.java`

- 注入 DataLogWriter
- 在 `submit()` 方法中调用 `dataLogWriter.logData(gameLog)`

### Step 5: 实现启动数据同步检查

**文件**: `src/main/java/com/gamelog/config/DataRecoveryRunner.java`

- 实现 `ApplicationRunner` 接口
- 读取最近 3 天的 JSON 日志文件
- 与数据库对比，找出未入库记录
- 批量插入缺失记录
- 记录同步结果日志

## Implementation Details

### 1. pom.xml 依赖
```xml
<!-- 排除默认 Logback -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <exclusions>
        <exclusion>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-logging</artifactId>
        </exclusion>
    </exclusions>
</dependency>

<!-- 添加 Log4j2 -->
<dependency>
    <groupId>org.apache.logging.log4j</groupId>
    <artifactId>log4j-spring-boot-starter</artifactId>
</dependency>
```

### 2. log4j2-spring.xml 配置
```xml
<RollingFile name="DataLog"
    fileName="logs/data/game-log.json"
    filePattern="logs/data/game-log-%d{yyyy-MM-dd}.json">
    <JsonTemplateLayout />
    <Policies>
        <TimeBasedTriggeringPolicy />
    </Policies>
    <DefaultRolloverStrategy max="7" />
</RollingFile>

<Logger name="DataLogger" level="info" additivity="false">
    <AppenderRef ref="DataLog" />
</Logger>
```

### 3. DataLogWriter.java
```java
@Slf4j
@Component
public class DataLogWriter {
    private final Logger dataLogger;

    public DataLogWriter() {
        this.dataLogger = LogManager.getLogger("DataLogger");
    }

    public void logData(GameLog gameLog) {
        // 异步写入，不阻塞
        dataLogger.info(JSON.toJSONString(gameLog));
    }
}
```

### 4. DataRecoveryRunner.java 逻辑
```
1. 获取 logs/data/ 目录下最近 3 天的文件
2. 逐行解析 JSON，记录唯一键集合
3. 查询数据库中已有记录
4. 计算差集 = 日志文件中有但数据库没有的
5. 批量插入缺失记录
6. 输出统计: 总数/成功/失败
```

## Assumptions & Decisions

1. **Log4j2 优于 Logback**: 性能更好，支持异步写入
2. **JSON 格式**: 便于解析和恢复
3. **保留周期**: 3 天足够（启动时能恢复）
4. **同步键**: 使用 `(gameName, player, action, playTime)` 组合键判重
5. **异步写入**: Log4j2 异步 Appender，不影响主流程性能
